# Q-A-DID 项目

## 项目背景

电商平台买家社区问答数据的差分分析（DID, Difference-in-Differences）。

数据字段：`auctionNumId`（商品编号）、`date`（提问时间）、`questionId`（问题ID）、`question`（提问内容）、`answer_date`（回答日期）、`answer_nickname`（回答者昵称，已脱敏）、`answer`（回答内容）、`时间是否在范围`（布尔值）

数据规模：62,559 条记录，99 个不同商品。

## 分析框架

处理节点：**2023年10月**，前后各4个季度（共8个季度）进行趋势对比。

| 标签 | 时间范围 |
|------|----------|
| pre_Q1 | 2022-10-01 ~ 2022-12-31 |
| pre_Q2 | 2023-01-01 ~ 2023-03-31 |
| pre_Q3 | 2023-04-01 ~ 2023-06-30 |
| pre_Q4 | 2023-07-01 ~ 2023-09-30 |
| post_Q1 | 2023-10-01 ~ 2023-12-31 |
| post_Q2 | 2024-01-01 ~ 2024-03-31 |
| post_Q3 | 2024-04-01 ~ 2024-06-30 |
| post_Q4 | 2024-07-01 ~ 2024-09-30 |

筛选条件：8个季度中每季度至少1条问答记录 → **66个商品**符合条件。

## 数据清洗流程

### 1. ID对应关系检查

- 问答.xlsx、商品详情.xlsx、评论数.xlsx 三个表的商品ID完全一致，共 **99个ID**，一一对应。
- 商品详情.xlsx 有830行（829条数据），因每个商品有多行（sku变体等），需group by。

### 2. 商品详情清洗

**原始列**：商品id | 标题 | skuid | sku | 价格 | 券后价格 | 店铺 | 店铺评分 | 服务 | 品牌 | 评论数

**清洗策略**：
- `价格`：同ID多行时，按价格取**中位数**行保留
- `skuid`、`sku`：删除（规格变体信息，无需）
- `券后价格`：删除（大量空值）
- `店铺评分`：解析JSON数组，拆分为**宝贝质量**、**物流速度**、**服务保障**三列
- `服务`、`品牌`：去除`[]`和`'`符号，仅保留逗号分隔
- `评论数`：将`1万+`、`6000+`等字符串转为**万为单位的数值**，`1万+`→`1`，`6000+`→`0.6`；列名改为`评论数（万）`
- `店铺评分`缺失值：2行（商品ID: 586953057988、666755837332），用**均值四舍五入**填充，宝贝质量/物流速度/服务保障三列均值均为 **4.9**

**清洗后列（10列）**：商品id | 标题 | 价格 | 店铺 | 宝贝质量 | 物流速度 | 服务保障 | 服务 | 品牌 | 评论数（万）

### 3. 品牌与百度指数匹配

- 百度指数CSV有44个品牌，与商品详情49个品牌进行匹配
- 匹配逻辑：子串匹配（大小写不敏感），如"L'oreal/欧莱雅"匹配"欧莱雅-化妆品"
- 匹配成功74个商品；未匹配25个（自然堂、Za/姬芮、Mistine等）
- 品牌写法统一为百度指数格式（如"欧莱雅-化妆品"）
- 百度指数列：已匹配的填入对应值，未匹配的9个品牌先留空

### 4. brand_awareness 特征

- 以66个商品在百度指数中的**去重唯一值**（40个）排序
- 取前1/3（约14个）对应品牌标记为 **1**，其余标记为 **0**
- 阈值：**59,757**（排名第14大的值）
- brand_awareness=1：53行；=0：37行；=null（未匹配）：9行
- 9个未匹配品牌后由用户手动补录了百度指数（仍按阈值判断）

### 5. 问答表筛选

- 按8季度窗口筛选（每季度≥1条）：66商品，**55,228条**记录
- 季度分布均匀（每季度约2,000~2,900条）

### 6. 左连接问答表与商品详情

- 问答_符合条件.xlsx **左连接**商品详情_清洗后.xlsx
- 连接键：`auctionNumId` = `商品id`
- 匹配成功：55,228行全部匹配

### 7. 最终特征构建

在窗口内数据（19,535行）上构建以下特征，并删除`时间是否在范围`列：

| 特征名 | 说明 | 生成规则 |
|--------|------|---------|
| Treat | 处理组标记 | 全部为0（对照组） |
| Quarter | 季度编号 | 1~8，对应8个季度窗口 |
| Post | 处理前后标记 | Q1~Q4=0（pre），Q5~Q8=1（post） |
| q_length | 提问字数 | `question`字段字符长度 |
| a_length | 回答字数 | `answer`字段字符长度 |
| ln_q_length | 提问字数对数 | `ln(q_length + 1)` |
| ln_a_length | 回答字数对数 | `ln(a_length + 1)` |
| price_moderate | 价格高中低 | 66个商品唯一价格去重排序，前1/3=1，其余=0；阈值=500 |

**价格阈值计算**：66个商品，64个去重价格，排序后前1/3（22个）≥500元

### 8. 已知数据问题修复

- **Quarter/Post空值**：日期字符串用`new Date()`解析存在UTC时区偏移，导致跨日。修复为从字符串直接提取`YYYY-MM-DD`年月日部分再比较。
- **ln公式修正**：a_length=1时ln(1)=0与ln(0)混淆，改为`ln(a_length + 1)`和`ln(q_length + 1)`。
- **ID文本格式**：`auctionNumId`和`questionId`转为文本字符串格式，避免Excel数值计算。
